# Product Requirements Document (PRD)

## 1. 文档信息

- 产品名称：跨境电商竞品与市场情报系统（Amazon Focus）
- 文档版本：v1.0
- 创建日期：2026-05-16
- 文档目标：围绕“持续追踪竞品策略、产品变化与营销动态”构建可执行的产品方案，重点覆盖竞品分析、市场监控、舆情追踪。

## 2. 背景与问题定义

跨境电商团队在亚马逊运营中普遍面临以下问题：

1. 竞品信息分散且更新快：新品上架、价格策略、广告动作、评价变化难以及时捕捉。
2. 市场变化感知滞后：类目趋势、关键词热度、季节性波动无法形成持续监控。
3. 舆情波动风险高：差评爆发、社媒负面讨论、KOL评价变化对销量影响大，但预警不及时。
4. 分析结果难落地：团队往往有数据但缺少“可执行建议”，无法形成明确动作优先级。

## 3. 产品目标

### 3.1 业务目标

1. 缩短竞品情报响应时间：从“周级”缩短到“天级/小时级”。
2. 提升选品与运营决策质量：形成有证据链的动作建议。
3. 降低舆情风险损失：在风险扩大前触发预警与处置建议。

### 3.2 产品目标

1. 建立统一的竞品、市场、舆情监控台。
2. 提供自动化追踪、变化检测、告警分发能力。
3. 输出标准化洞察报告与行动建议（而非仅展示原始数据）。

## 4. 目标用户与使用场景

### 4.1 目标用户

1. 品类经理：关注类目机会、竞品格局、上新方向。
2. 运营经理：关注价格、促销、广告、评价变化并快速响应。
3. 增长/投放团队：关注关键词竞争度、广告策略变化、流量趋势。
4. 品牌与客服团队：关注口碑趋势、负面舆情、售后风险。
5. 管理层：关注周/月度竞争态势与经营风险摘要。

### 4.2 核心场景

1. 每日晨会：自动汇总昨日竞品变化与风险事件。
2. 活动期监控：Prime Day、黑五等期间实时跟踪价格与广告动作。
3. 新品立项：基于类目趋势、竞品空白、用户痛点形成机会评估。
4. 舆情危机：差评激增或社媒负面扩散时快速定位原因与处置建议。

## 5. 需求范围

### 5.1 核心模块

1. 竞品分析中心
2. 市场监控中心
3. 舆情监测中心
4. 预警与通知中心
5. 报告与策略建议中心

### 5.2 非目标（MVP阶段不做）

1. 自动执行广告投放或自动改价。
2. 覆盖所有平台（先聚焦 Amazon US，可扩展 EU/JP）。
3. 完整 CRM/ERP 功能替代。

## 6. 详细需求

## 6.1 竞品分析中心（重点）

### 6.1.1 竞品池管理

1. 支持按 ASIN、品牌、关键词导入竞品。
2. 支持分层管理：核心竞品、次级竞品、观察竞品。
3. 支持竞品标签：价格带、定位、卖点、材质、场景、人群。

### 6.1.2 竞品变化追踪

1. 商品页信息追踪：标题、主图、A+、五点描述、变体、配送时效。
2. 商业策略追踪：价格、Coupon、Deal、捆绑、库存状态。
3. 评价口碑追踪：评分、评论数量、差评率、关键词主题变化。
4. 排名表现追踪：BSR、关键词自然排名、类目位置变化。

### 6.1.3 竞品策略识别

1. 价格策略识别：短期促销、长期降价、价格带迁移。
2. 内容策略识别：主图风格变化、卖点调整、场景重构。
3. 口碑运营识别：评论增长异常、评分修复节奏。
4. 广告动作识别：关键词覆盖扩张、品牌词拦截倾向（基于可得数据推断）。

### 6.1.4 对比分析与输出

1. 支持“我方ASIN vs 竞品组”多维对比看板。
2. 自动生成差距分析：价格、内容、口碑、流量入口、转化要素。
3. 给出建议动作：优先级、预估影响、执行成本、风险提示。

## 6.2 市场监控中心

### 6.2.1 类目趋势监控

1. 追踪目标类目规模趋势、季节波动、头部集中度变化。
2. 监控子类目机会：增速快但竞争相对可控的细分方向。

### 6.2.2 关键词与需求信号

1. 监控核心关键词热度、竞争强度、点击成本趋势（接入可得工具数据）。
2. 识别新兴需求词、长尾机会词、风险词（投诉/合规敏感词）。

### 6.2.3 新品与品牌动态

1. 监控新上架ASIN数量、增长速度、类目渗透情况。
2. 监控新品牌入场和头部品牌动作（上新节奏、价格策略切换）。

## 6.3 舆情监测中心

### 6.3.1 站内舆情（Amazon）

1. 评论主题聚类：质量、物流、包装、功能、售后等。
2. 情感趋势分析：正/中/负占比及变化斜率。
3. 风险识别：差评激增、同类问题高频复现、星级突降。

### 6.3.2 站外舆情（可扩展）

1. 覆盖社媒/社区/评测站的品牌与竞品声量监控。
2. 识别传播节点：高影响账号、热点帖子、负面扩散路径。
3. 形成“舆情事件卡片”：事件内容、传播规模、影响评估、建议应对。

## 6.4 预警与通知中心

1. 告警规则配置：阈值、波动率、连续天数、组合条件。
2. 告警分级：P1（紧急）、P2（高）、P3（中）。
3. 通知渠道：站内消息、邮件、企业IM（如钉钉/飞书/Slack）。
4. 告警去噪：相似事件合并、冷却时间、误报标注与学习。

## 6.5 报告与策略建议中心

1. 日报：昨日变化摘要 + 今日优先动作。
2. 周报：竞品策略变化、类目趋势、舆情复盘、下周建议。
3. 专题报告：活动期复盘、新品立项评估、竞品深度拆解。
4. 支持模板化导出（Markdown/PDF/Slides）。

## 7. 数据与指标设计

## 7.1 数据来源（MVP建议）

1. Amazon 公开商品页与评论数据（合规采集/第三方合规数据服务）。
2. 第三方电商情报工具数据（关键词、排名、流量估算）。
3. 社媒/社区公开数据（用于站外舆情扩展）。
4. 企业内部数据（可选）：我方广告、销量、退货、客服工单。

## 7.2 核心指标体系

1. 竞品变化捕获率：被成功识别的关键变化事件占比。
2. 告警有效率：被业务确认“有动作价值”的告警占比。
3. 响应时效：从事件发生到告警触达的时间。
4. 决策采纳率：建议动作被执行的比例。
5. 业务结果关联：采纳建议后核心指标改善幅度（CTR/CVR/评分等）。

## 8. 功能优先级（MoSCoW）

### Must Have（MVP）

1. 竞品池管理与核心字段追踪（价格、评分、评论、排名、内容变化）。
2. 关键告警（价格突变、评分骤降、差评激增、竞品上新）。
3. 日报/周报自动生成。
4. Amazon US 单站点支持。

### Should Have

1. 策略识别（价格战、内容换代、评论运营节奏）。
2. 关键词趋势和新词发现。
3. 舆情主题聚类与风险分级。

### Could Have

1. 多站点联动（US/EU/JP）。
2. 站外舆情传播链分析。
3. 建议动作效果回流学习。

## 9. 里程碑规划

1. M1（第1-2周）：需求冻结、数据源与技术方案确定、指标定义。
2. M2（第3-6周）：MVP开发（竞品追踪+预警+日报周报）。
3. M3（第7-8周）：试运行与规则调优（降噪、阈值校准）。
4. M4（第9-10周）：上线与运营SOP固化、首轮复盘。

## 10. 风险与约束

1. 数据合规风险：需严格遵守平台政策与数据使用规范。
2. 数据质量风险：页面结构变化、异常值、时效延迟。
3. 告警噪音风险：规则过敏导致“告警疲劳”。
4. 解释可信度风险：策略识别需区分“事实”与“推断”。

## 11. 验收标准（MVP）

1. 可稳定追踪不少于 50 个核心竞品ASIN，日更成功率 >= 95%。
2. 关键事件告警时延 <= 6 小时（可逐步优化到小时级）。
3. 每周输出可直接用于业务会议的标准报告（无需二次加工）。
4. 业务团队对告警“有用”评价占比 >= 70%（试运行阶段目标）。

## 12. 附录：建议的MVP页面结构

1. 首页总览：昨日关键变化、风险告警、待处理建议。
2. 竞品看板：竞品池、ASIN详情、变化时间线、差距分析。
3. 市场看板：类目趋势、关键词机会、品牌动态。
4. 舆情看板：情感趋势、问题主题、风险事件。
5. 报告中心：日报、周报、专题报告、导出与订阅。
