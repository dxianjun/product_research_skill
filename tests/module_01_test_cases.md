# 模块 01 测试用例：竞品池管理

## 功能范围

- 竞品新增（ASIN、品牌、关键词、分层、标签）
- 重复 ASIN 防重
- ASIN 格式校验
- 分层更新
- 分层查询

## 测试用例清单

1. `TC-01` 新增竞品成功  
输入：合法 ASIN、品牌、tier、关键词/标签（含重复项）  
预期：成功写入；ASIN 归一化为大写；关键词与标签去重

2. `TC-02` 重复 ASIN 新增失败  
输入：同一 ASIN 连续新增两次  
预期：第二次抛出 `ValueError`

3. `TC-03` 非法 ASIN 拒绝  
输入：非 10 位字母数字 ASIN  
预期：抛出 `ValueError`

4. `TC-04` 更新分层成功  
输入：已存在 ASIN，tier 从 `watch` 改为 `secondary`  
预期：更新成功，返回最新 tier

5. `TC-05` 更新不存在 ASIN 失败  
输入：不存在的 ASIN  
预期：抛出 `KeyError`

6. `TC-06` 按分层查询  
输入：2 个 `core` + 1 个 `watch`  
预期：查询 `core` 返回 2 条

## 用例合理性校验

1. 覆盖主路径：`TC-01` 覆盖核心成功流程。
2. 覆盖关键异常：`TC-02/03/05` 对防重、格式、不存在对象三个高频风险点进行验证。
3. 覆盖核心业务语义：`TC-04/06` 验证“竞品分层管理”这一核心业务能力。
4. 覆盖边界处理：`TC-01` 中包含关键词与标签去重，验证输入清洗逻辑。
5. 与 PRD 对齐：对应 PRD 中“竞品池管理（导入、分层、标签）”的 MVP 要求。
