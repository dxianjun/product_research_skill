# 模块 04 测试用例：告警通知中心

## 功能范围

- 通知抽象层（站内、邮件）
- 告警等级订阅与路由分发
- 幂等控制（同告警避免重复发送）
- 异常配置校验（不支持渠道）

## 测试用例清单

1. `TC-01` 单渠道订阅分发  
输入：P1 订阅站内渠道 + 1 条 P1 告警  
预期：发送 1 条站内通知

2. `TC-02` 多渠道并发分发  
输入：P1 同时订阅站内与邮件  
预期：发送 2 条通知，分别落到两个渠道

3. `TC-03` 无订阅不发送  
输入：告警等级无订阅  
预期：返回 0 条通知

4. `TC-04` 幂等去重生效  
输入：同一告警重复分发两次  
预期：第一次发送成功，第二次不重复发送

5. `TC-05` 非法渠道订阅失败  
输入：订阅未注册渠道（如 sms）  
预期：抛出 `ValueError`

## 用例合理性校验

1. 覆盖主流程：单渠道与多渠道分发路径完整。
2. 覆盖关键约束：无订阅不发送与非法渠道拦截避免误发。
3. 覆盖稳定性要求：幂等去重减少重复通知噪音。
4. 与 PRD 对齐：对应“通知渠道、分发策略、去噪控制”要求。
