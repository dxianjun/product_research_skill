# 模块 03 测试用例：预警规则引擎

## 功能范围

- 阈值规则触发（increase/decrease）
- 告警分级（P1/P2/P3）与排序
- 去噪冷却（同 ASIN + 同 cooldown key 去重）
- 规则异常校验

## 测试用例清单

1. `TC-01` 降价阈值触发  
输入：价格下降幅度大于阈值  
预期：生成 1 条告警，等级符合规则

2. `TC-02` 低于阈值不触发  
输入：评分下降幅度小于阈值  
预期：不产生告警

3. `TC-03` 去噪冷却生效  
输入：同一 ASIN 同一类事件连续触发两次  
预期：仅输出 1 条告警

4. `TC-04` 分级排序正确  
输入：同时触发 P1 与 P3  
预期：返回顺序中 P1 在前

5. `TC-05` 非法规则方向拦截  
输入：direction 非 increase/decrease  
预期：抛出 `ValueError`

## 用例合理性校验

1. 规则主路径覆盖：触发与不触发均验证。
2. 关键业务规则覆盖：分级与去噪是告警系统核心，已覆盖。
3. 防错误配置覆盖：非法规则参数验证可降低线上风险。
4. 与 PRD 对齐：对应“预警规则、分级、去噪”MVP 要求。
